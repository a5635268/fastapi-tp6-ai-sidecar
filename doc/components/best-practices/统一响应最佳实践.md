# 统一响应最佳实践

本文档介绍如何在 fastapi-tp6 项目中使用统一响应格式，确保 API 返回数据的一致性。

## 1. 响应格式标准

所有 API 返回统一的 JSON 格式：

```json
{
  "code": 0,
  "msg": "调用成功",
  "time": 1707475200,
  "data": {}
}
```

分页响应增加分页信息：

```json
{
  "code": 0,
  "msg": "调用成功",
  "time": 1707475200,
  "data": [],
  "total": 100,
  "page": 1,
  "page_size": 10,
  "pages": 10
}
```

---

## 2. 基础使用规范

### 2.1 成功响应

```python
from app.core.response import ResponseBuilder

# 返回数据
return ResponseBuilder.success(data=user_info, msg="获取成功")

# 无数据返回
return ResponseBuilder.success(msg="操作成功")
```

### 2.2 错误响应

```python
# 通用错误
return ResponseBuilder.error(code=1, msg="操作失败")

# 参数验证错误
return ResponseBuilder.validate_error(msg="用户名不能为空")

# 未授权
return ResponseBuilder.unauthorized(msg="Token 已过期")

# 资源不存在
return ResponseBuilder.not_found(msg="用户不存在")
```

### 2.3 分页响应

```python
from app.utils.page_util import PageUtil

# 方式一：使用 PageUtil 配合
page_result = await PageUtil.paginate(queryset, page=1, page_size=10)
return PageUtil.to_paginated_response(page_result, msg="查询成功")

# 方式二：直接使用 ResponseBuilder
return ResponseBuilder.paginated(
    data=items,
    total=100,
    page=1,
    page_size=10,
    msg="查询成功"
)
```

## 3. 异常处理规范

### 3.1 使用 ApiException

业务异常使用 `ApiException` 抛出，全局异常处理器会自动转换：

```python
from app.core.response import ApiException

# 抛出异常（会被自动转换为统一响应）
raise ApiException(code=12, msg="资源不存在")

# 带附加数据
raise ApiException(code=1, msg="操作失败", data={"errors": ["字段错误"]})
```

### 3.2 异常转换结果

抛出的异常会转换为：

```json
{
  "code": 12,
  "msg": "资源不存在",
  "time": 1707475200,
  "data": null
}
```

## 4. 错误码使用规范

### 4.1 预定义错误码

| 错误码 | 场景 | HTTP 状态 |
|--------|------|-----------|
| 0 | 成功 | 200 |
| 1 | 通用失败 | 500 |
| 2 | 认证失败 | 401 |
| 3 | 限流 | 429 |
| 6 | 数据为空 | 400 |
| 12 | 资源不存在 | 404 |
| 21 | 参数错误 | 400 |

### 4.2 自定义错误码

业务特定错误码建议使用 3000+ 范围：

```python
from app.core.response import ErrorCodeManager

# 注册业务错误码
ErrorCodeManager.register(
    code=3001,
    msg="用户积分不足",
    http_status=400
)

# 使用自定义错误码
raise ApiException(code=3001)
```

## 5. 路由层最佳实践

### 5.1 标准路由写法

```python
from fastapi import APIRouter, Request
from app.core.response import ResponseBuilder, ApiException
from app.services.user import UserService

router = APIRouter(prefix="/users", tags=["用户"])

@router.get("/{user_id}")
async def get_user(request: Request, user_id: int):
    """获取用户信息"""
    user = await UserService.get_by_id(user_id)
    if not user:
        raise ApiException(code=12, msg="用户不存在")
    return ResponseBuilder.success(data=user, msg="获取成功")

@router.post("/")
async def create_user(request: Request, user_data: dict):
    """创建用户"""
    user = await UserService.create(user_data)
    return ResponseBuilder.success(data=user, msg="创建成功")
```

### 5.2 避免直接返回字典

```python
# ❌ 不推荐：直接返回字典
return {"user": user}

# ✅ 推荐：使用统一响应
return ResponseBuilder.success(data=user)
```

## 6. 服务层最佳实践

### 6.1 异常抛出而非返回错误

```python
# ❌ 不推荐：返回错误字典
def get_user(user_id):
    user = db.query(user_id)
    if not user:
        return {"error": "用户不存在"}
    return user

# ✅ 推荐：抛出异常
async def get_user(user_id):
    user = await User.get_or_none(id=user_id)
    if not user:
        raise ApiException(code=12, msg="用户不存在")
    return user
```

### 6.2 服务层只返回数据

```python
# ✅ 服务层只返回数据，路由层包装响应
async def get_user_list(page, page_size):
    return await PageUtil.paginate(User.all(), page, page_size)

# 路由层
@router.get("/")
async def list_users(request: Request, page: int = 1, page_size: int = 10):
    result = await UserService.get_user_list(page, page_size)
    return PageUtil.to_paginated_response(result)
```

## 7. 响应头与特殊响应

### 7.1 限流响应

```python
return ResponseBuilder.too_many_requests(
    msg="请求过于频繁",
    headers={
        "X-RateLimit-Limit": "100",
        "X-RateLimit-Remaining": "0",
        "Retry-After": "60"
    }
)
```

### 7.2 流式响应

```python
async def generate_stream():
    for chunk in data_chunks:
        yield chunk

return ResponseBuilder.streaming(
    content=generate_stream(),
    media_type="text/event-stream"
)
```

## 8. 常见问题

**Q: 如何处理第三方接口返回的不规范数据？**

在服务层转换为统一格式：

```python
async def call_external_api():
    external_data = await fetch_external()
    # 转换为统一格式返回
    return external_data.get('result')
```

**Q: 如何区分业务错误和系统错误？**

- 业务错误：使用 `ApiException` 抛出，code 范围 1-999
- 系统错误：返回 HTTP 500，code 为 1 或特定系统错误码

**Q: 响应中的 time 字段有什么用？**

用于前端调试和日志追踪，表示服务器响应时间戳。
