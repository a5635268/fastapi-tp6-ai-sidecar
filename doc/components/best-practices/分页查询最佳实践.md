# 分页查询最佳实践

本文档介绍如何在 fastapi-tp6 项目中使用 `PageUtil` 进行分页查询，与 `ResponseBuilder` 配合返回标准分页响应。

## 1. 分页场景选择

### 1.1 数据库分页 vs 内存分页

| 场景 | 推荐方式 | 说明 |
|------|----------|------|
| 数据库查询 | `PageUtil.paginate` | 直接数据库分页，性能最优 |
| 已有列表数据 | `PageUtil.get_page_obj` | 内存切片分页 |
| 第三方数据 | `PageUtil.get_page_obj` | 无数据库控制权时使用 |

---

## 2. ORM 分页查询

### 2.1 基础用法

```python
from app.utils.page_util import PageUtil
from app.models.user import User

# Tortoise ORM 分页
page_result = await PageUtil.paginate(
    queryset=User.all(),
    page=1,
    page_size=20,
)
```

### 2.2 带条件查询

```python
# 带筛选条件
page_result = await PageUtil.paginate(
    queryset=User.filter(status='active'),
    page=1,
    page_size=20,
)

# 排序
page_result = await PageUtil.paginate(
    queryset=User.all().order_by('-created_at'),
    page=1,
    page_size=20,
)

# 带关联查询
page_result = await PageUtil.paginate(
    queryset=User.all().prefetch_related('profile'),
    page=1,
    page_size=20,
)
```

### 2.3 返回结果结构

```python
{
    "data": [...],      # 当前页数据（Model 对象列表）
    "total": 100,       # 总记录数
    "page": 1,          # 当前页码
    "page_size": 20,    # 每页数量
    "pages": 5          # 总页数
}
```

## 3. 内存切片分页

### 3.1 基础用法

```python
# 对已有列表进行分页
all_items = [{'id': i, 'name': f'item_{i}'} for i in range(100)]

page_result = PageUtil.get_page_obj(
    data_list=all_items,
    page=2,
    page_size=10,
)
```

### 3.2 适用场景

- 第三方 API 返回的完整列表
- 内存中已加载的数据
- 需要二次处理的数据

## 4. 返回分页响应

### 4.1 使用 PageUtil.to_paginated_response

```python
from app.core.response import PaginatedResponse

# 将字典转换为 PaginatedResponse
response = PageUtil.to_paginated_response(
    page_result,
    msg="获取成功"
)
```

### 4.2 使用 ResponseBuilder.paginated

```python
from app.core.response import ResponseBuilder

# 直接构建分页响应
return ResponseBuilder.paginated(
    data=items,
    total=100,
    page=1,
    page_size=10,
    msg="查询成功"
)
```

## 5. 路由层完整示例

### 5.1 标准分页路由

```python
from fastapi import APIRouter, Request, Query
from app.utils.page_util import PageUtil
from app.models.user import User

router = APIRouter(prefix="/users", tags=["用户"])

@router.get("/")
async def list_users(
    request: Request,
    page: int = Query(1, ge=1, description="页码"),
    page_size: int = Query(10, ge=1, le=100, description="每页数量"),
    status: str = Query(None, description="状态筛选"),
):
    """用户列表（分页）"""
    queryset = User.all()
    
    if status:
        queryset = queryset.filter(status=status)
    
    queryset = queryset.order_by('-created_at')
    
    page_result = await PageUtil.paginate(queryset, page, page_size)
    
    return PageUtil.to_paginated_response(page_result, msg="获取成功")
```

### 5.2 参数校验

使用 FastAPI Query 参数校验：

```python
page: int = Query(1, ge=1)          # 页码 >= 1
page_size: int = Query(10, ge=1, le=100)  # 每页数量 1-100
```

## 6. 服务层分页封装

### 6.1 服务层返回分页结果

```python
class UserService:
    @staticmethod
    async def get_user_list(page, page_size, filters=None):
        queryset = User.all()
        
        if filters:
            if filters.get('status'):
                queryset = queryset.filter(status=filters['status'])
            if filters.get('keyword'):
                queryset = queryset.filter(name__contains=filters['keyword'])
        
        return await PageUtil.paginate(queryset, page, page_size)
```

### 6.2 路由层调用

```python
@router.get("/")
async def list_users(request: Request, page: int = 1, page_size: int = 10):
    result = await UserService.get_user_list(page, page_size)
    return PageUtil.to_paginated_response(result)
```

## 7. 数据序列化

### 7.1 Model 对象自动序列化

Tortoise ORM Model 对象会自动序列化为 JSON：

```python
# Model 对象列表
page_result = await PageUtil.paginate(User.all(), 1, 10)

# 自动序列化
{
    "data": [
        {"id": 1, "name": "Alice", "status": "active"},
        {"id": 2, "name": "Bob", "status": "active"}
    ],
    "total": 100,
    ...
}
```

### 7.2 自定义序列化

```python
# 需要自定义字段时，先转换为字典
users = await User.all()
user_dicts = [await u.to_dict() for u in users]

page_result = PageUtil.get_page_obj(user_dicts, 1, 10)
```

## 8. 分页与缓存

### 8.1 分页接口缓存

```python
from app.annotations import ApiCache

@ApiCache(namespace='user_list', expire_seconds=60)
async def list_users(request: Request, page: int = 1, page_size: int = 10):
    # 缓存包含分页参数，不同页码独立缓存
    return await UserService.get_user_list(page, page_size)
```

### 8.2 缓存失效

```python
from app.annotations import ApiCacheEvict

@ApiCacheEvict(namespace_prefixes=['user'])
async def update_user(request: Request, user_id: int):
    # 更新用户后清理所有 user 相关缓存（包括列表）
    return await UserService.update(user_id)
```

## 9. 常见问题

**Q: PageUtil.paginate 与 ResponseBuilder.paginated 有什么区别？**

- `PageUtil.paginate`：执行数据库查询，返回字典
- `ResponseBuilder.paginated`：构建响应模型，不执行查询
- 配合使用：`PageUtil.paginate` → `PageUtil.to_paginated_response`

**Q: 如何获取总页数？**

分页结果中的 `pages` 字段就是总页数，自动计算。

**Q: 大数据量分页如何优化？**

- 使用数据库索引
- 避免 `offset` 过大（可考虑游标分页）
- 限制最大 `page_size`

**Q: 如何实现游标分页？**

当前 `PageUtil` 使用传统 offset/limit 分页。游标分页需要自行实现或扩展。
