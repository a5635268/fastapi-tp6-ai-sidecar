# 缓存策略最佳实践

本文档介绍如何在 fastapi-tp6 项目中使用 `ApiCache` 和 `ApiCacheEvict` 注解，合理配置缓存策略以提升系统性能。

## 1. 缓存适用场景

### 1.1 适合缓存的场景

- **热点数据查询**：频繁访问的配置、字典数据
- **公共数据**：不依赖用户状态的列表、详情
- **计算密集型数据**：聚合统计、复杂查询结果
- **第三方接口响应**：减少外部调用

### 1.2 不适合缓存的场景

- **用户个性化数据**：用户私有信息、权限数据
- **实时性要求高**：消息、通知、状态
- **写操作频繁**：数据变更频繁导致缓存失效成本高

---

## 2. 基础配置规范

### 2.1 命名空间设计

命名空间应按业务模块划分：

```python
# ✅ 推荐：按模块划分
@ApiCache(namespace='users', expire_seconds=60)       # 用户模块
@ApiCache(namespace='articles', expire_seconds=300)   # 文章模块
@ApiCache(namespace='configs', expire_seconds=600)    # 配置模块

# ❌ 不推荐：过于泛化
@ApiCache(namespace='api', expire_seconds=60)
```

### 2.2 过期时间配置

根据数据更新频率设置合理过期时间：

| 数据类型 | 推荐过期时间 |
|----------|--------------|
| 热点数据 | 60s |
| 字典数据 | 300s |
| 配置数据 | 600s |
| 第三方数据 | 根据接口特性 |

## 3. 读场景缓存策略

### 3.1 公共数据缓存

```python
from app.annotations import ApiCache

@ApiCache(namespace='dict', expire_seconds=300)
async def get_user_status_dict(request: Request):
    """获取用户状态字典（公共数据）"""
    return {'active': '活跃', 'inactive': '停用'}
```

### 3.2 详情查询缓存

```python
@ApiCache(namespace='article_detail', expire_seconds=60)
async def get_article(request: Request, article_id: int):
    """文章详情（热点数据）"""
    article = await ArticleService.get_by_id(article_id)
    return article
```

### 3.3 用户隔离缓存

```python
@ApiCache(namespace='user_profile', expire_seconds=60, vary_by_user=True)
async def get_user_profile(request: Request):
    """用户个人资料（需登录）"""
    user = RequestContext.get_current_user()
    profile = await UserService.get_profile(user['id'])
    return profile
```

## 4. 写场景缓存失效

### 4.1 单命名空间失效

```python
from app.annotations import ApiCacheEvict

@ApiCacheEvict(namespaces=['article_detail'])
async def update_article(request: Request, article_id: int, data: dict):
    """更新文章后清理详情缓存"""
    return await ArticleService.update(article_id, data)
```

### 4.2 多命名空间失效

```python
@ApiCacheEvict(namespaces=['article_detail', 'article_list', 'article_stats'])
async def delete_article(request: Request, article_id: int):
    """删除文章后清理多个相关缓存"""
    return await ArticleService.delete(article_id)
```

### 4.3 按前缀批量失效

```python
@ApiCacheEvict(namespace_prefixes=['article'])
async def batch_update_articles(request: Request, updates: list):
    """批量更新后清理所有 article 相关缓存"""
    # 清理：article_detail, article_list, article_stats 等
    return await ArticleService.batch_update(updates)
```

## 5. 缓存降级处理

### 5.1 Redis 不可用时自动降级

```python
@ApiCache(namespace='hot_data', expire_seconds=60)
async def get_hot_data(request: Request):
    # Redis 不可用时自动执行原函数
    # 不会阻塞业务
    return await DataService.get_hot_data()
```

### 5.2 降级日志

Redis 不可用时输出警告日志：

```
WARNING: 当前应用未初始化 Redis 连接，跳过接口缓存
INFO: Redis 未初始化，缓存降级：直接执行原函数
```

## 6. 缓存管理

### 6.1 手动清理缓存

```python
from app.annotations import ApiCacheManager
from app.core.redis import RedisUtil

redis = RedisUtil.get_redis()

# 清理特定命名空间
await ApiCacheManager.clear_namespace(redis, 'users')

# 清理多个命名空间
await ApiCacheManager.clear_namespaces(redis, ['users', 'articles'])

# 按前缀清理
await ApiCacheManager.clear_namespace_prefix(redis, 'user')

# 清理所有接口缓存（慎用）
await ApiCacheManager.clear_all(redis)
```

### 6.2 缓存命中监控

缓存命中时响应头会添加 `X-Api-Cache: HIT`：

```python
# 可通过响应头判断缓存是否命中
response.headers.get('X-Api-Cache')  # 'HIT' 表示命中
```

## 7. 组合使用示例

### 7.1 读写分离缓存配置

```python
# 查询接口：启用缓存
@ApiCache(namespace='user', expire_seconds=60)
async def get_user(request: Request, user_id: int):
    return await UserService.get_by_id(user_id)

# 更新接口：清理缓存
@ApiCacheEvict(namespaces=['user'])
async def update_user(request: Request, user_id: int, data: dict):
    return await UserService.update(user_id, data)
```

### 7.2 与其他注解组合

```python
from app.annotations import ApiCache, ApiRateLimit, ApiRateLimitPreset

@ApiRateLimit(namespace='api', preset=ApiRateLimitPreset.USER_COMMON_MUTATION)
@ApiCache(namespace='user_list', expire_seconds=60)
async def get_user_list(request: Request, page: int = 1):
    return await UserService.get_list(page)
```

## 8. 常见问题

**Q: 缓存键是如何生成的？**

基于请求方法、路径、查询参数、请求体哈希生成唯一键。`vary_by_user=True` 时加入用户标识。

**Q: 如何判断缓存是否生效？**

查看响应头 `X-Api-Cache: HIT` 表示命中。

**Q: 缓存失效后数据不一致怎么办？**

确保写操作后正确清理相关命名空间的缓存。

**Q: 大数据量如何缓存？**

不建议缓存大数据量响应。如需缓存，考虑：
- 缩短过期时间
- 按条件分片缓存
- 使用异步后台任务预热
