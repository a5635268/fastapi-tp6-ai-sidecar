这是一个很常见的开发场景，我来帮你实现一键部署的方案。

## 方案一：VS Code 插件 + 脚本（推荐）

### 1. 安装 SFTP 插件
在 VS Code 中安装 **SFTP** 插件（作者：Natizyskunk）

### 2. 配置 SFTP
在项目根目录创建 `.vscode/sftp.json`：
```json
{
    "name": "My Server",
    "host": "your-server-ip",
    "protocol": "sftp",
    "port": 22,
    "username": "your-username",
    "remotePath": "/path/to/your/app",
    "uploadOnSave": false,
    "ignore": [
        ".vscode",
        ".git",
        "__pycache__",
        "*.pyc",
        ".env"
    ]
}
```

### 3. 创建部署脚本 `deploy.sh`
```bash
#!/bin/bash

# 配置参数
SERVER="your-username@your-server-ip"
REMOTE_PATH="/path/to/your/app"
CONTAINER_NAME="your-container-name"

echo "🚀 开始部署..."

# 1. 同步代码（使用 rsync，排除不需要的文件）
rsync -avz --delete \
    --exclude '__pycache__' \
    --exclude '*.pyc' \
    --exclude '.git' \
    --exclude '.vscode' \
    --exclude '.env' \
    --exclude 'venv' \
    ./ $SERVER:$REMOTE_PATH/

# 2. 远程重启容器
ssh $SERVER "cd $REMOTE_PATH && docker-compose restart $CONTAINER_NAME"

echo "✅ 部署完成！"
```

### 4. 配置 VS Code 任务
在 `.vscode/tasks.json` 中添加：
```json
{
    "version": "2.0.0",
    "tasks": [
        {
            "label": "deploy",
            "type": "shell",
            "command": "bash",
            "args": ["deploy.sh"],
            "group": {
                "kind": "build",
                "isDefault": true
            },
            "presentation": {
                "reveal": "always",
                "panel": "new"
            }
        }
    ]
}
```

现在你只需要按 `Ctrl+Shift+B` 就能一键部署！

## 方案二：Makefile（更简洁）

在项目根目录创建 `Makefile`：
```makefile
# 配置
SERVER = your-username@your-server-ip
REMOTE_PATH = /path/to/your/app
CONTAINER = your-container-name

# 颜色输出
GREEN = \033[0;32m
NC = \033[0m

deploy:
	@echo "$(GREEN)🚀 同步代码...$(NC)"
	rsync -avz --delete \
		--exclude '__pycache__' \
		--exclude '*.pyc' \
		--exclude '.git' \
		--exclude '.vscode' \
		--exclude '.env' \
		./ $(SERVER):$(REMOTE_PATH)/
	@echo "$(GREEN)🔄 重启容器...$(NC)"
	ssh $(SERVER) "cd $(REMOTE_PATH) && docker-compose restart $(CONTAINER)"
	@echo "$(GREEN)✅ 部署完成！$(NC)"

# 快速重启（只重启，不同步代码）
restart:
	ssh $(SERVER) "cd $(REMOTE_PATH) && docker-compose restart $(CONTAINER)"

# 查看日志
logs:
	ssh $(SERVER) "cd $(REMOTE_PATH) && docker-compose logs -f $(CONTAINER)"
```

使用方式：
```bash
make deploy    # 部署
make restart   # 只重启
make logs      # 查看日志
```

## 方案三：使用 rsync + ssh 别名

### 1. 配置 SSH 别名（~/.ssh/config）
```
Host myserver
    HostName your-server-ip
    User your-username
    Port 22
    IdentityFile ~/.ssh/id_rsa
```

### 2. 创建 Python 部署脚本 `deploy.py`
```python
#!/usr/bin/env python3
import subprocess
import sys
import os

def run_command(cmd, capture=False):
    """执行命令并打印输出"""
    print(f"执行: {cmd}")
    result = subprocess.run(cmd, shell=True, capture_output=capture, text=True)
    if result.returncode != 0:
        if capture:
            print(result.stderr)
        sys.exit(1)
    return result

def main():
    # 配置
    SERVER = "myserver"  # 使用 SSH 别名
    REMOTE_PATH = "/path/to/your/app"
    CONTAINER = "your-container-name"
    
    print("🚀 开始部署...")
    
    # 1. 同步代码
    rsync_cmd = f"""
    rsync -avz --delete \
        --exclude '__pycache__' \
        --exclude '*.pyc' \
        --exclude '.git' \
        --exclude '.vscode' \
        --exclude '.env' \
        --exclude 'venv' \
        ./ {SERVER}:{REMOTE_PATH}/
    """
    run_command(rsync_cmd)
    
    # 2. 重启容器
    restart_cmd = f'ssh {SERVER} "cd {REMOTE_PATH} && docker-compose restart {CONTAINER}"'
    run_command(restart_cmd)
    
    print("✅ 部署完成！")

if __name__ == "__main__":
    main()
```

使用：`python deploy.py`

## 方案四：使用 Docker 的热重载（开发环境）

如果只是开发调试，可以使用 **Docker 热重载**，代码修改后自动重启：

### docker-compose.yml
```yaml
version: '3.8'

services:
  app:
    build: .
    container_name: your-app
    volumes:
      # 挂载代码目录，实现热更新
      - ./app:/app
    environment:
      - PYTHONUNBUFFERED=1
    command: uvicorn main:app --host 0.0.0.0 --port 8000 --reload
    ports:
      - "8000:8000"
```

然后在服务器上：
```bash
# 同步代码后，容器会自动重载（因为使用了 --reload）
rsync -avz --exclude '__pycache__' ./ app/ your-server:/path/to/app/
```

## 推荐配置总结

**最推荐：方案一（VS Code SFTP + 脚本）**

优势：
- ✅ 可视化操作，右键即可上传
- ✅ 支持自动保存上传（可配置）
- ✅ 一键部署，无需切换到终端
- ✅ 支持文件对比和冲突处理

**最佳实践：**
1. 使用 Git 管理代码
2. 在服务器上配置 Git hook，push 后自动部署
3. 结合 CI/CD（GitHub Actions/GitLab CI）

这样你只需要 `git push`，就能自动完成部署，真正做到一键部署！
