“父子分段”模式下，将父块设置为**“全文”**，并将子块检索规则设置为按 `\n\n` 切分、最大长度 512 字符。

在 Dify 的 API 中，父块选择“全文”对应的核心参数是 `"parent_mode": "full-doc"`。当父块为全文时，原本的 `segmentation` 节点就会自动被 Dify 识别为**子块**的切分规则。

你需要将 API 请求中的 `doc_form` 和 `process_rule` 部分修改为以下结构：

```json
  "doc_form": "hierarchical_model",
  "process_rule": {
    "mode": "custom",
    "rules": {
      "pre_processing_rules": [
        {
          "id": "remove_extra_spaces",
          "enabled": true
        },
        {
          "id": "remove_urls_emails",
          "enabled": false
        }
      ],
      "parent_mode": "full-doc",
      "segmentation": {
        "separator": "\n\n",
        "max_tokens": 512,
        "chunk_overlap": 0 
      }
    }
  }
```

### 核心参数映射说明：
1. **`"doc_form": "hierarchical_model"`**：开启父子分段模式。
2. **`"parent_mode": "full-doc"`**：对应截图中的父块用作上下文选择 **“全文”**（如果是“段落”则为 `"paragraph"`）。
3. **`"segmentation"` 块**：因为父块是全文，这里的规则直接作用于截图下方的**“子块用于检索”**。
   * `"separator": "\n\n"`：对应你的分段标识符。
   * `"max_tokens": 512`：对应你的分段最大长度 512 characters。
